## 分析框架

### WBS分类（最小化）

**新项目：**
```
├── 1.0 MVP核心功能
├── 2.0 必要集成
├── 3.0 关键基础设施
└── 4.0 最低非功能需求
```

**现有项目：**
```
├── 1.0 直接影响范围
├── 2.0 必需更改
├── 3.0 依赖链
└── 4.0 风险缓解
```

### 复杂度评估

| 尺寸 | 点数 | 描述 |
|------|------|------|
| XS | 1-3 | 简单CRUD+验证 |
| S | 4-8 | 业务逻辑+基本集成 |
| M | 9-13 | 复杂规则+API集成 |
| L | 14-20 | 架构更改+性能优化 |
| XL | 21+ | 领域重设计+可扩展性 |

### 最小化过滤

对每个WBS元素检验：
1. MVP业务关键？→ Y/N
2. 无法作为MVP后增强？→ Y/N
3. 需立即架构决策？→ Y/N
4. 省略会阻塞核心功能？→ Y/N
